SolDiff Review

The workspace is a polished Next.js landing/demo app for a Solana upgrade auditor, but it does not yet contain the advertised CLI, SDK, fetch/decompile pipeline, or any on-chain program code. The review therefore focuses on product completeness, correctness, and frontend quality rather than Solana contract security.

High - Correctness

Advertised Solana pipeline is not implemented

The repository claims an open-source CLI, SDK, archival RPC fetcher, decompiler, diff engine, and governance integrations in README.md, but the actual workspace contains only a Next.js UI with static demo components and no backend or CLI package. That makes the current product materially incomplete and easy to misrepresent as production-ready.

Fix: either implement the missing core packages and runtime pipeline, or explicitly re-scope the docs/UI to a prototype and label simulated outputs as demo-only.

Evidence: README.md, package.json, app/components/DemoSection.tsx
Medium - Correctness

Animation timers are not cleaned up

Hero.tsx schedules delayed terminal-line timers, and DemoSection.tsx starts staged loading intervals plus a delayed selection update, but neither effect returns cleanup for those timers. If the component unmounts or a new selection starts, the old timers can still fire and update state.

Fix: store timeout and interval IDs in refs, clear them from the effect cleanup, and cancel any pending staged loading work when a new program is selected.

Evidence: app/components/Hero.tsx, app/components/DemoSection.tsx
